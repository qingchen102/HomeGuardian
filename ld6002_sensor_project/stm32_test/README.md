# STM32 测试固件 — LD6002 传感器验证

## 用途

在交付 ESP32-C5 正式代码前，用 STM32F103C8T6 (Blue Pill) 快速验证 LD6002 传感器：

- 验证硬件连接正确性
- 验证 1382400bps 高速串口通信稳定性
- 验证 TinyFrame 协议解析正确性
- 通过串口助手直观查看 JSON 输出数据

## 硬件连接

```
┌──────────────────┐              ┌──────────────────────┐
│  LD6002 雷达模组  │              │  STM32F103C8T6       │
│                  │              │  (Blue Pill)         │
│  1: 3V3 ─────────┼──────┬───────┤  3.3V (AMS1117输出)  │
│  2: GND ─────────┼──────┼───────┤  GND                 │
│  3: P19 ─────────┼──────┤       │  GND  ←─ 接GND!     │
│  7: TX0 ─────────┼──────┼───────┤  PA10 (USART1 RX)    │
│  8: RX0 ─────────┼──────┼───────┤  PA9  (USART1 TX)    │
│                  │      │       │                      │
│                  │      │       │  PA2  (USART2 TX) ───┼──> USB-TTL RX (PC)
│                  │      │       │  GND ────────────────┼──> USB-TTL GND
└──────────────────┘      │       └──────────────────────┘
                          │
                   ┌──────┴──────────────┐
                   │ 5V USB ──> AMS1117  │
                   │         3.3V ≥1A    │
                   │  + 470μF + 100nF    │
                   │  (电源滤波很重要!)    │
                   └─────────────────────┘
```

## 编译

### Keil MDK

1. 创建新项目 → Device: STM32F103C8
2. 添加所有 `.c` 文件到项目
3. 添加 Include Paths: `Inc/`
4. 需要 STM32F1 HAL 库 (可从 STM32CubeMX 生成)
5. 编译 → 下载

### STM32CubeIDE

1. 基于 STM32F103C8 创建新项目
2. 配置 USART1: 1382400 bps, 8N1 (需要手动设置波特率)
3. 配置 USART2: 115200 bps, 8N1
4. 替换 `Core/Src/main.c` 和添加其他源文件
5. 编译 → 下载

### USART1 波特率配置 (USART1 on APB2, 72MHz)

```c
huart1.Init.BaudRate = 1382400;

// HAL 内部计算:
// DIV = 72,000,000 / (16 × 1,382,400) = 3.2552
// Mantissa = 3, Fraction = 4 (0.25×16)
// 实际 = 1,384,615 bps, 误差 0.16% ✓ (< 2%)
```

## 测试步骤

1. 按接线图连接所有硬件
2. **确认 LD6002 P19 已接 GND** (否则模组不进正常模式)
3. USB-TTL 模块插入 PC，打开串口助手
   - 波特率: **115200**
   - 数据位: 8
   - 停止位: 1
   - 校验: None
4. 上电，串口助手应显示:
   ```
   [INIT] LD6002 Sensor Test Firmware
   [INIT] STM32F103C8T6 @ 72MHz
   [INIT] System boot OK
   [INIT] LD6002 driver initialized (USART1, 1382400 bps)
   [INIT] Sensor settling... (~60s)
   ```
5. **等待约 1 分钟**，期间传感器正在建立探测
6. 传感器稳定后，每 200ms 输出一行 JSON:
   ```json
   {"ts":60001,"br":18.5,"hr":72.3,"bp":1.23,"hp":5.67,"tp":3.45,"rng":85.0,"pres":1,"alert":0,"msg":""}
   ```
7. 测试报警:
   - 手离开传感器探测区域 → 应输出 `"pres":0,"alert":4,"msg":"bed_leave"`
   - 回到探测区域，心率应恢复正常

## 常见问题

### 无任何输出?
- STM32 时钟是否正确 (HSE 8MHz × 9 PLL = 72MHz)?
- USART2 TX (PA2) 线序是否正确?

### 有初始化日志但无 JSON?
- LD6002 P19 是否接了 GND?
- LD6002 供电是否 ≥1A? (USB口输出可能不足)
- 电源纹波是否过大? (增加 470μF 电容)

### JSON 中数值全为 0 或乱码?
- 电源问题! 强烈建议在 LD6002 3V3 和 GND 之间并联 470μF 电解 + 100nF 陶瓷电容
- USART1 波特率是否准确? (检查 HSE 晶振是否为 8MHz)

### 心率数值偏差大?
- 传感器安装位置是否正确? (床头1m高, 45°倾斜, 距离1.5m内)
- 检测区域内是否只有一个人?
- 被检测者是否保持静止? (大动作会导致测量暂停)
