## 接口约定

### 传感器模块 → 主控模块

**物理层**: UART, 115200 bps, 8N1, 无流控

**数据格式**: JSON, 每帧以换行符 `\n` 结尾

**更新频率**: 5Hz (每200ms一帧)

### JSON 字段说明

```json
{
    "ts":    1234567,    // uint32, 毫秒时间戳 (模组上电后累计)
    "br":    18.5,       // float,  呼吸率 (breaths per minute), 范围 9-48
    "hr":    72.3,       // float,  心率 (beats per minute), 范围 60-150
    "bp":    1.23,       // float,  呼吸相位
    "hp":    5.67,       // float,  心跳相位
    "tp":    3.45,       // float,  总相位
    "rng":   85.0,       // float,  目标距离 (cm)
    "pres":  1,          // uint8,  人体存在标志 (1=有人, 0=无人)
    "alert": 0,          // uint8,  报警标志位 (见下表)
    "msg":   ""          // string, 报警描述 (多个告警以逗号分隔)
}
```

### 报警标志位 (alert 字段)

| 位 | 值 | 含义 | 说明 |
|---|----|------|------|
| 0 | 0x01 | `heart_low` | 心率 < 50 bpm |
| 1 | 0x02 | `heart_high` | 心率 > 120 bpm |
| 2 | 0x04 | `bed_leave` | 离床/无人检测 |
| 3 | 0x08 | `sensor_err` | 传感器超时异常 |
| 4 | 0x10 | `settling` | 传感器建立中 (上电60s内) |

**报警处理逻辑**:
- 仅 `settling`: 正常建立过程，无需主动报警
- 包含 `heart_low` 或 `heart_high`: 立即触发蜂鸣器 + 推送通知 + 云端记录
- 包含 `bed_leave`: 仅推送通知 (老人可能只是暂时起身)
- 包含 `sensor_err`: 本地蜂鸣器提示检查设备
- 报警解除 (`alert` 从非零变为0): 关闭蜂鸣器，记录恢复日志

## 协作方集成步骤

1. 将 `esp32c5_production/components/ld6002_sensor/` 复制到主控项目
2. 在主控的 `CMakeLists.txt` 中包含该组件
3. 实现 `on_data` 和 `on_alert` 两个回调函数
4. 调用 `ld6002_sensor_start(on_data, on_alert)` 启动传感器
5. 详细说明见 `docs/INTEGRATION_GUIDE.md`

## 引脚占用

传感器模块使用 ESP32-C5 的以下引脚 (可通过 sdkconfig 修改):

| 引脚 | 功能 | 方向 |
|------|------|------|
| GPIO4 | UART TX → LD6002 RX | 输出 |
| GPIO5 | UART RX ← LD6002 TX | 输入 |

**协作方注意**: 这些引脚不可用于其他功能。
